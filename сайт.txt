Руководство по запуску системы "Администрирование сувенирного магазина"
Данный проект состоит из бэкенда (на Python + Flask + MySQL) и десктопного фронтенда (на PyQt6). Ниже приведена пошаговая инструкция по развёртыванию и запуску.

 Требования
Python 3.8+
MySQL 5.7+ (или MariaDB)
Библиотеки Python: flask, flask-sqlalchemy, flask-cors, pymysql, PyQt6, requests
Установка зависимостей
# Установите необходимые пакеты
pip install flask flask-sqlalchemy flask-cors pymysql PyQt6 requests
Настройка базы данных
Создайте базу данных в MySQL:
sql
 CREATE DATABASE souvenirs_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci; 
Убедитесь, что пользователь MySQL имеет права на эту базу (по умолчанию используется root:password).
При необходимости отредактируйте строку подключения в файле api.py:
python
 app.config["SQLALCHEMY_DATABASE_URI"] = "mysql+pymysql://ваш_логин:ваш_пароль@localhost/souvenirs_db?charset=utf8mb4" 
▶️ Порядок запуска
1. Запустите бэкенд
Перейдите в папку back и запустите API-сервер:

bash
 cd back
python api.py 
Сервер будет доступен по адресу: http://127.0.0.1:5000

При первом запуске автоматически создастся пользователь admin с паролем admin.

2. (Опционально) Заполните базу тестовыми данными
Если вы хотите сразу работать с готовыми товарами, поставщиками и заказами:

bash
 cd back
python init_db.py 
Подтвердите действие вводом y.

⚠️ Скрипт удалит все существующие данные, если база уже содержит записи.

3. Запустите фронтенд
Перейдите в папку desktop и запустите приложение:

bash
 cd desktop
python front.py 
4. Войдите в систему
Логин: admin
Пароль: admin


Очистка базы данных
Для полной очистки данных (например, перед новым тестированием):

bash
 cd back
python clear_db.py 
Подтвердите действие вводом y.

Структура проекта
yuraproject/
├── back/
│   ├── api.py          # Бэкенд-сервер
│   ├── init_db.py      # Скрипт инициализации тестовых данных
│   └── clear_db.py     # Скрипт полной очистки БД
├── desktop/
│   └── front.py        # Десктопный клиент
Советы
Убедитесь, что MySQL-сервер запущен перед запуском api.py.
Если вы изменяете структуру моделей, может потребоваться пересоздать таблицы (очистка + инициализация).
Все операции (добавление/редактирование/удаление) автоматически обновляют данные в интерфейсе — кнопка "Обновить" нужна только для ручного обновления.
